# Commit Often

## Overview

Commit at natural stopping points throughout a session — not only at the end. A natural stopping point is any moment the work is coherent and in a working state: a subtask is complete, a test passes, or the context is about to shift to a different area. Use judgment on cadence unless the user specifies otherwise.

Good stopping points: a named subtask is done, a test goes green, work is about to move to a different area of the codebase, or a risky structural change is coming next.

Never leave completed, working changes uncommitted at the end of a session.

## Output

No structured output. This skill shapes commit behavior throughout the session.

## Hard Stop

None. Commit guidance is advisory — the agent uses judgment unless the user specifies a cadence.
